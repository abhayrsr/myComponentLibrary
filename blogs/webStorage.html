What is Web Storage?

Web storage also known as DOM storage, is one of the feature in HTML5. With web storage, the applications can store the data on the client site and can be manipulated accordingly. The data is mainly stored in key-value form where the key is a string and the value can be of any data type like strings, numbers, objects converted to JSON strings.

Why web storage was introduced?

Web storage was introduced as a part of HTML5 because there was some limitations in data storing mechanism like cookies(We'll discuss about this later in the blog).

  • Storage Capacity: Web storage provides developers an ease of storing more data on the client side as compared to cookies. It is easy to store data upto 4mb per domain with web storage.

  • Performance Enhancement: Web storage is designed to be faster and efficient for data storage and retrieval compared to cookies.

  • Security: Web storage is more secure than cookies. There are two strong reasons for that - first, with every HTTP request the data is not sent to the server automatically which limits the data exposure to any third party. Second, web storage follows the same origin policy that means the data cannot be accessed from different origin or domain keeping the security intact.

    • Web storage provides two objects to store the data on browser:
    window.localStorage
    window.sessionStorage

    Types of Web Storage

    There are two types of web storage:

    1. Session Storage

    It stores the data for one session that means when the browser tab is closed, the data in sessionStorage is cleared. In order to clear the sessionStorage object manually use sessionStorage.clear().

    To store data in session storage, use sessionStorage.setItem(key, value) method in JavaScript.

    To retrieve data, you can use sessionStorage.getItem(key)

    sessionStorage.setItem("name", "Peter");
    const name = sessionStorage.getItem("name");
    console.log(name);

    Output: Peter

    2. Local Storage

    It stores the data locally within the browser. Even if you close the browser tab, the data will still remain unless you remove it manually using localStorage.clear().

    To store data in session storage, use localStorage.setItem(key, value) method in JavaScript.

    To retrieve data, you can use localStorage.getItem(key)

    localStorage.setItem("name", "Peter");
    const name = localStorage.getItem("name");
    console.log(name);

    Output: Peter

    Let's discuss a little bit about cookies

    Cookies are the small piece of data sent by the web server to user's browser and is stored in user's device. With every request to the server a cookie is sent back from the user's device.

    We know what are the limitations of using the cookies. But there are certain benefits attached to the usage of cookies some of which are listed below:

  • Cross-Domain sharing: Web storage is restricted to same-origin policy but in case of cross-domain sharing cookies are a nice fit configured to work across different domain, allowing cross-domain authentication and sharing of limited data.

  • Compatibility: Cookies are well supported by almost all web browsers which makes it quite compatible to use.

    • But still it is recommended to use Web storage over cookies because of its high storage capacity and enhanced security. No doubt, cookies have their place but it all depends on the requirement of your web application.
